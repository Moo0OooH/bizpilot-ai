# BizPilot AI — Strategic Alignment Implementation Checklist v1.6

## Purpose

This checklist converts the v1.6 strategic alignment into implementation-ready work without deleting or weakening existing v1.4/v1.5 standards.

## A. Product Positioning

- [ ] Update landing/dashboard/demo copy to lead with quote recovery.
- [ ] Avoid primary language such as generic AI platform, CRM, operating system, or automation suite.
- [ ] Use cleaning-first examples in MVP demos.
- [ ] Keep long-term platform vision only in strategic/background sections.

## B. Magic Moment

- [ ] Add first-use sample cleaning lead state.
- [ ] Include urgency/status tag.
- [ ] Include AI-style summary.
- [ ] Include suggested reply draft.
- [ ] Include follow-up risk/next action.
- [ ] Include CTA to review reply and share quote link.
- [ ] Clearly label sample/demo content.

## C. Security Hardening

- [ ] Verify explicit GRANTs.
- [ ] Harden public quote insert policies.
- [ ] Validate public field writability at database/helper level.
- [ ] Reject hidden or invalid fields.
- [ ] Add abuse protection.
- [ ] Add honeypot/spam friction.
- [ ] Wire RLS tests into package scripts.
- [ ] Add safe error and logging review.
- [ ] Add AI metadata cleanup.

## D. Operational Calm UX

- [ ] QA dashboard at 100% zoom.
- [ ] Reduce visual noise.
- [ ] Improve card hierarchy.
- [ ] Make primary next action obvious.
- [ ] Improve empty states.
- [ ] Keep AI visual treatment restrained.
- [ ] Ensure status/follow-up visibility.

## E. Founder-Led GTM

- [x] Create Founder CRM spreadsheet/template.
- [x] Track contacted businesses.
- [x] Track objections and owner language.
- [x] Track demos and follow-ups.
- [x] Track willingness to pay.
- [x] Track pilot retention notes.
- [x] Use concierge onboarding for first customers.

Evidence: `artifacts/phase18/BizPilot_Phase18_Founder_CRM_Template.xlsx` and `docs/operations/BIZPILOT_PHASE_18_PILOT_OPERATING_GUIDE_v1.0.md`.

## F. AI Constraints

- [ ] AI drafts are concise and operational.
- [ ] AI does not send messages.
- [ ] AI does not invent prices or availability.
- [ ] AI outputs are structured/validated where used by the app.
- [ ] Owner remains final decision maker.

## G. Validation Gate

Do not expand verticals or major feature categories until:

- [ ] 3+ paying/payment-ready cleaning businesses exist.
- [ ] Repeat weekly usage is visible.
- [ ] Retention is healthy.
- [ ] Onboarding is stable.
- [ ] At least one testimonial or strong qualitative proof exists.
- [ ] Quote-to-response workflow is clearly useful.

## Definition of Done

The project is strategically aligned when a new coding agent, designer, or founder can read the docs and clearly understand that the current mission is not to build a broad AI platform, but to validate a secure, focused, cleaning-first quote recovery product that owners repeatedly use.
